HERRAMIENTAS DE TECNOLOGÍAS BACKEND

JAVA SCRIPT – EXPRESS.JS


Definición Es un framework gratuito y de código abierto, se basa en un marco back-end de Node.js minimalista y rápido, con herramientas sólidas para desarrollar aplicaciones web y móviles, con funcionalidades como el enrutamiento, opciones para gestionar sesiones y cookies, etc.
Características principales
  1. Es minimalista, provocando que sea una herramienta liviana.
  2. La secuencia de sus comandos permite la reducción de gastos.
  3. Proporciona una herramienta de interfaz de línea de comando (CLI).
  4. Implementa middlewares, es decir tiene acceso al objeto de solicitud (req) y al objeto de respuesta (res).
  5. Pertenece a la comunidad de código abierto.
Ventajas Desventajas
Permite a los desarrolladores usar un solo lenguaje para Back-end y front-end Es difícil alojar los marcos de back-end de JavaScript con bases de datos MySQL
Sus funciones integradas permiten el acceso a DOM y Hoop, etc. Existe malinterpretaciones entre las devoluciones de llamadas y la redacción del código en la devolución de la llamada.(problema que en la version actual, no es tan común)
Brinda un sistema de enrutamiento y funciones simplificadas para ampliar el marco La naturaleza de la falta de supervisión en las tecnologías back-end de javaScript

Tipo Licencia : Licencia MIT, licencia de software libre permisiva.
Ejemplo de uso : Esta herramienta fue usada para la implementación de PayPal, Fox sports, IBM entre otras.
Página Oficial : https://expressjs.com/es/

PYTHON – FLASK


Definición Es un módulo de Python, se clasifica en un microframework que permite desarrollar aplicaciones web.

Características principales
  1. Servidor de desarrollo y depurador.
  2. Soporte para cookies seguras (las sesiones son de lado cliente)
  3. Usa plantillas Jinja (sirven para generar documentos HTML)
  4. Compatibilidad del motor de aplicaciones de Google.
  5. Es un framework minimalista
Ventajas Desventajas
Incluye un servidor web de desarrollo, que permite correr las apps en la web, sin necesidad de un adicional. Las capas de acceso a la BD no se encuentran completamente desarrolladas.
Es óptima para construir contenido estático. Necesita bibliotecas adicionales.
Tiene un depurador y soporte para pruebas unitarias. Utiliza un código de baja calidad

Tipo Licencia : Licencia BSD, son de software libre permisiva con un mínimo de restricciones.
Ejemplo de uso : Las aplicaciones que hacen uso de este microframework son: Pinterest y linkedln.
Página Oficial : https://flask.palletsprojects.com/en/2.2.x/

PHP – SYMFONY


Definición
Es un framework completo que se basa en el patrón Modelo Vista Controlador, fue construido con componentes independientes que fueron creados por el mismo Symfony.

Características principales
  1. Es compatible con los SO de Windows y Unix-like.
  2. Utiliza la programación orientada a objetos.
  3. Usa templates y layouts para la presentación.
  4. Permite la interacción con AJAX (permite crear aplicaciones web asíncronas).
  5. Es de instalación y configuración fácil.
Ventajas Desventajas
Permite el desarrollo desde sitios web sencillos hasta macro proyectos. Requiere de dedicación para comprender las habilidades que contiene.
Permite una formación continua, debido al soporte de la comunidad. No cuenta con bibliotecas avanzadas.
Es de fácil mantenimiento. Debido a las dificultades que presenta, muy pocos desarrolladores lo toman en cuentan para elaborar los proyectos.

Tipo Licencia : Usa la licencia MIT, es de software libre permisiva.
Ejemplo de uso: Se puede tomar como ejemplo a Yahoo bookmarks, que permite almacenar los marcadores web.
Página Oficial : https://symfony.com/

SPRING BOOT


Definición
Es un framework que permite la creación de aplicaciones de servidor basadas en java.

Características principales
  1. Consta de funciones para producción, como métricas, controles de estado y configuración externalizada.
  2. No necesita requisitos para configuración XML.
  3. Ofrece procesamiento por lotes.
  4. Tiene base de datos integradas y en memoria.
  5. Crea aplicaciones Spring independientes.
Ventajas Desventajas
Ofrece un flujo de trabajo simple. Permite la acumulación de capas, dificultando la depuración.
Ofrece pruebas unitarias y de integración. Consta de funcionalidades complejas.Necesita demasiado tiempo de Startup.
Ofrece una interfaz accesible. Necesita demasiado tiempo de Startup.

Tipo Licencia : Fue lanzado bajo la licencia Apache2.0
Ejemplo de uso : El aplicativo de Zillow implemento esta herramienta, se dedica a bienes raíces.
Página Oficial : https://spring.io/projects/spring-boot
